# 用户行为漏斗分析

为产品和运营团队提供用户行为漏斗建模、转化率分析、行为路径追踪和优化策略的完整工作流。

## 核心漏斗模型

### 标准转化漏斗

```
曝光 → 点击 → 落地页 → 注册 → 激活 → 留存 → 付费 → 复购
```

每一步的转化率 = 当前步骤用户数 / 上一步骤用户数 × 100%

### 游戏行业典型漏斗

```
广告曝光 → 点击下载 → 安装完成 → 首次启动 → 完成新手引导 → 首次付费 → 持续活跃
```

行业基准参考：
- 曝光→点击: 1-3% (CTR)
- 点击→安装: 30-50%
- 安装→启动: 70-85%
- 启动→完成新手: 50-70%
- 新手→首充: 2-5%
- 首充→月活跃付费: 40-60%

### 电商/出海典型漏斗

```
广告曝光 → 落地页访问 → 商品浏览 → 加入购物车 → 发起结算 → 完成支付
```

行业基准参考：
- 曝光→访问: 1-5%
- 访问→浏览商品: 40-60%
- 浏览→加购: 8-15%
- 加购→结算: 30-50%
- 结算→支付: 60-80%

## 关键指标体系

### 转化指标
- CVR (Conversion Rate): 整体转化率
- 步骤转化率: 各步骤间的转化比例
- 漏斗完成率: 从首步到末步的整体转化
- 平均转化时长: 用户完成整个漏斗的平均耗时

### 行为指标
- 页面停留时长: 用户在各页面的平均停留
- 跳出率 (Bounce Rate): 仅访问单页即离开的比例
- 退出率 (Exit Rate): 从特定页面离开的比例
- 点击深度: 用户平均点击/交互次数

### 计算公式

```
整体转化率 = 最终转化用户数 / 漏斗入口用户数 × 100%
步骤流失率 = 1 - 步骤转化率
加权转化率 = Σ(各渠道转化率 × 各渠道流量占比)
漏斗效率 = 实际转化率 / 理论最优转化率
```

## 分析工作流

### 1. 漏斗定义与埋点

定义漏斗前需明确：
- 业务目标 (注册、付费、分享等)
- 关键步骤节点
- 每步的判定条件和时间窗口

埋点方案设计：
- 客户端埋点: 页面浏览(PV)、按钮点击、滑动、输入
- 服务端埋点: 接口调用、订单创建、支付回调
- 全埋点: 自动采集所有交互事件

事件命名规范：
```
{对象}_{动作}_{属性}
示例: page_view, button_click, order_submit, payment_success
```

### 2. 漏斗可视化与诊断

输出内容：
- 漏斗图: 各步骤用户数和转化率的可视化
- 流失节点标注: 转化率低于基准的步骤高亮
- 时间趋势: 各步骤转化率随时间的变化
- 分群对比: 不同用户群的漏斗差异

诊断维度：
- 按渠道: 自然流量 vs 付费流量 vs 社交裂变
- 按设备: iOS vs Android vs Web
- 按地区: 不同国家/地区的转化差异
- 按时段: 工作日 vs 周末，高峰 vs 低谷
- 按用户属性: 新用户 vs 老用户，VIP vs 普通

### 3. 行为路径分析

用户旅程地图：
- 主路径: 最常见的用户行为序列
- 分支路径: 偏离主路径的行为模式
- 回环路径: 用户反复操作的节点
- 终止节点: 用户最终离开的页面/步骤

分析方法：
- 桑基图 (Sankey Diagram): 展示用户流向
- 热力图: 页面点击/注意力分布
- 会话回放: 还原用户真实操作过程
- 归因分析: 识别影响转化的关键因素

### 4. A/B测试框架

测试流程：
1. 提出假设 (如: "简化注册流程可提升转化率10%")
2. 设计实验方案 (对照组 vs 实验组)
3. 确定样本量和测试周期
4. 分流执行 (通常50/50或90/10)
5. 数据收集与统计检验
6. 结论与上线决策

样本量计算：
```
最小样本量 ≈ 16 × σ² / δ²
σ = 指标标准差
δ = 最小可检测差异 (MDE)
置信度通常取95%, 统计功效取80%
```

常见测试场景：
- 落地页布局/文案/CTA按钮
- 注册流程步骤数
- 定价策略和促销方案
- 推送通知时间和内容
- 新手引导流程

### 5. 转化率优化策略

#### 入口优化 (曝光→点击)
- 优化广告素材和文案
- 精准定向目标人群
- 测试不同广告格式 (视频/图片/轮播)

#### 落地页优化 (点击→注册)
- 首屏突出核心价值主张
- 减少表单字段 (仅保留必要信息)
- 添加社交证明 (用户评价、数据背书)
- 提供多种注册方式 (手机/邮箱/第三方登录)
- 页面加载速度 < 3秒

#### 激活优化 (注册→活跃)
- 设计引导式新手体验
- 快速展示产品核心价值 (Aha Moment)
- 个性化推荐内容
- 适时推送提醒

#### 付费优化 (活跃→付费)
- 设置低门槛首次付费 (如1元体验)
- 限时优惠制造紧迫感
- 展示付费前后对比
- 简化支付流程

## 报告模板

漏斗分析报告应包含：
1. 漏斗概览: 各步骤用户数、转化率、环比变化
2. 流失分析: 流失最严重的步骤及原因假设
3. 分群对比: 不同维度下的漏斗表现差异
4. 行为路径: 主要用户路径和异常路径
5. 优化建议: 针对流失节点的具体改进方案
6. A/B测试计划: 待验证的优化假设和实验设计
7. 预期收益: 优化后的转化率提升预估和GMV/收入影响
